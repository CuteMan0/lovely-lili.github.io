了解SSD1306(I2C)

了解I2C

飞利浦（NXP）于1980提出I2C串行总线，使用多主从架构，为同步半双工通信方式
常见速率：100kbps、300kbps、3.4mbps

I2C的数据格式


当检测到START信号，主机输出8bit——从机地址(7bit)和读写操作(1bit)，第9位为ACK应答信号，之后的数据以 N*(Byte+ACK) 的格式传输，发出STOP信号表示传输结束

写数据的方式参考
void Write_IIC_Byte(unsigned char IIC_Byte)
{
    unsigned char i;
    unsigned char m,data;
    data=IIC_Byte;
    OLED_SCLK_Clr();
    for(i=0;i<8;i++)	//1Byte data	
    {
    m=data;
    m=m&0x80;         //取首位
    if(m==0x80)
    {OLED_SDIN_Set();}
    else OLED_SDIN_Clr();
    data=data<<1;           //将下一位数据推至首位，准备下次读取
    OLED_SCLK_Set();    //读SDA的数据
    OLED_SCLK_Clr();
    }
}

I2C的START和STOP信号


在SCL为高电平期间读到SDA下降沿为START信号
在SCL为高电平期间读到SDA上升沿为STOP信号

I2C的ACK应答信号


接受设备检测到START信号，在接下来的8个CLK中，SDA进行一次8bit的数据传输，若接收到数据，则在第9个CLK对SDA拉低，将其视作一次ACK应答信号。

I2C的数据有效性


在传输期间，SCL为高电平时数据有效，数据的切换只能在SCL为低电平时进行。
