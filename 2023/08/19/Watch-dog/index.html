WDG

2023/8/19 MCU

看门狗

其实就是一个定时器，在程序跑飞/死循环时复位

IWDG （宠物狗）


一个 12bit 的递减计数器，计数器值为 0 时，产生一个 IWDG_RESET 复位信号

相关时间计算
Tiwdg = (4*2^prv) / LSI * rlv (s)
注：prv ---- 预分频器值（8bit）     rlv ---- 重载寄存器值
E.g：LSI = 40K， prv = 256，rlv = 625，则Tiwdg = 4s

WWDG （警犬）

一个 7bit 的递减计数器到固定值 0X40 时不喂狗，产生复位，这个值叫窗口的下限，固定不可变
计数器值在减到某一个数之前喂狗的话也会产生复位，这个值叫窗口的上限，上限值由用户独立设置

相关时间计算
Twwdg = Tpclk1 x 4096 x 2^wdgtb x (T[5:0] + 1) (ms)
注：Tpclk1 ---- APB1时钟周期， 2^wdgtb ---- 预分频器（4bit） 
    T[5:0] ---- 我所理解：T6位置0时，WWDG复位，所以上限值的T6位都是置1的，上下限差值为0x00-0x3F
E.g：Tpclk1 = 1/36M， wdgtb = 0, T[5:0] = 0x00, 则Twwdg = 113μs

区别

IWDG 因为采用 LSI 独立于系统时钟，故可监测硬件错误
WWDG 用于监测软件逻辑错误，且能在中断中进行抢救。
